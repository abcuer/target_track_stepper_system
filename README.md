# 基于分布式架构的火源视觉追踪与伺服报警系统

本系统荣获 **睿抗机器人开发者大赛 (RAICOM) 全国总决赛三等奖**。项目采用“大脑（OrangePi 5 Plus）+ 小脑（TI-MSPM0）”的分布式架构，实现了从火源特征提取、空间实时追踪到硬件级联动报警的完整闭环控制。



## 🚀 项目核心特性

* **分布式架构设计**：由 OrangePi 5 Plus 承担高负荷视觉算力，TI-MSPM0 负责高实时性电机控制，通过自研协议高效协同。
* **高可靠 FSM 通信协议**：自主设计基于**有限状态机 (FSM)** 的 UART 通信协议，集成双帧头校验与异或校验机制，支撑 115200bps 高频数据零丢包传输。
* **视觉伺服与动态锁定**：建立图像像素误差至空间转动角度的线性映射模型，采用**位置式 PID 控制思想**，实现平滑、无震荡的火源咬合轨迹。
* **高精度运动执行**：基于**定时器中断分发机制**驱动 CNC Shield V3，实现双轴步进电机脉冲同步控制。
* **智能联动报警**：集成**动态容差到位检测算法** [代码]，在云台精准锁定目标后自动触发硬件级声光报警逻辑。

## 🛠️ 技术栈

* **上位机 (Vision)**: Python, OpenCV, NumPy, PySerial
* **下位机 (Control)**: C, MSPM0 SDK, GPTIMER (PWM), GPIO, UART
* **硬件平台**: OrangePi 5 Plus, TI-MSPM0G3507, 42步进电机, CNC Shield V3

## 📂 文件结构说明

### 上位机 (Python)
* `detect.py`: 火源识别核心算法。基于 **HSV 色彩空间建模**与形态学滤波实现特征提取。
* `main.py`: 视觉伺服主程序。负责获取相机流、计算角度增量并下发指令。
* `serial_send.py`: 协议封装层。实现数据限幅、映射及**异或校验帧**构建。

### 下位机 (C/MSPM0)
* `usart.c`: 核心协议解析层。利用 **FSM 状态机**解析视觉端下发的控制指令。
* `motor.c`: 电机驱动层。封装了角度转步数算法及**定时器中断脉冲分发**逻辑。
* `motor.h`: 硬件抽象层。定义 `MOTOR_t` 对象结构体，实现硬件资源的抽象化封装。

## ⚙️ 控制算法原理

### 1. 视觉特征提取
算法通过 `cv2.inRange` 提取特定高温火焰色域，结合多边形轮廓拟合解算火源质心坐标 $(c_x, c_y)$。

### 2. 位置式伺服计算
系统根据当前水平视场角（FOV）建立映射，计算像素偏差 $\Delta x$ 对角度的影响：
$$Angle_{next} = Angle_{current} + \Delta x \cdot K_p$$
其中 $K_p$ 为根据视场角与分辨率动态调整的比例系数。



### 3. 到位检测逻辑
下位机实时监测步进电机已走步数 `count` 与目标步数 `tar` 的差值。为过滤机械震动干扰，引入动态容差带（Dead Zone）判断 [代码]：

```c
// 核心到位判定逻辑
if (abs(StepX.count - StepX.tar) <= 5) { // 进入容差死区范围，视为锁定成功
    SoundLightRun(); // 联动触发预警
}
```
